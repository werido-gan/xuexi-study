# 🧠 企业级 Prompt 设计（防幻觉 + 强约束）

> 目标：让大模型**稳定、可控、不乱编**，适用于 Agent / RAG / 生产系统

---

# 🚨 一、问题本质（为什么会幻觉）

```text
LLM 本质是“概率生成器”，不是数据库
```

👉 所以会：

- 编造事实（Hallucination）
    
- 过度自信
    
- 忽略不确定性
    

---

# 🎯 二、企业级 Prompt 设计目标

```text
1. 可控（输出稳定）
2. 可约束（不乱回答）
3. 可验证（结果可信）
4. 可扩展（适配Agent/RAG）
```

---

# 🧩 三、核心设计原则（必须掌握）

---

## 📌 原则1：明确边界（最重要）

```text
如果不知道 → 必须说不知道
```

---

### ✅ 示例

```text
如果无法从提供的信息中得出答案，请明确回答：
“无法根据现有信息回答”
```

---

## 📌 原则2：强制基于上下文

```text
只能使用提供的数据
```

---

### ✅ 示例

```text
你只能基于以下内容回答，不允许使用外部知识：
{context}
```

---

## 📌 原则3：结构化输出

```text
减少自由发挥空间
```

---

### ✅ 示例

```json
{
  "answer": "...",
  "confidence": "...",
  "source": "..."
}
```

---

## 📌 原则4：显式推理（可选）

```text
让模型“展示思考”
```

👉 但生产中建议：

```text
隐藏 Chain-of-Thought，只保留结果
```

---

## 📌 原则5：约束行为

```text
明确禁止行为
```

---

### 示例

```text
禁止：
- 编造数据
- 猜测答案
- 使用未提供信息
```

---

# 🔒 四、防幻觉 Prompt 模板（RAG 专用）

---

## 📌 标准模板（直接可用）

```text
你是一个严谨的AI助手。

请严格遵守以下规则：

1. 只能基于“已提供的上下文”回答问题
2. 如果上下文中没有答案，请回答：
   “无法根据现有信息回答”
3. 不允许编造、猜测或补充信息
4. 回答必须简洁、准确

【上下文】
{context}

【问题】
{question}

【回答】
```

---

## 📌 强化版（企业推荐）

```text
你是企业级问答系统的一部分，请严格遵守：

- 数据优先级：上下文 > 你的知识
- 不允许使用训练数据中的隐含知识
- 如果信息不足，必须返回“无答案”

输出格式：

{
  "answer": "...",
  "confidence": "high/medium/low",
  "reason": "依据说明"
}
```

# 🤖 五、Agent Prompt 设计（防失控）

---

## 📌 核心问题

Agent 容易：

- 无限调用工具
    
- 错误决策
    
- 乱执行
    

---

## 📌 强约束 Prompt

```text
你是一个Agent，请遵守：

1. 每次只能执行一个Action
2. 如果信息足够，直接回答
3. 最多执行 3 次工具调用
4. 如果工具结果无用，停止

格式：

Thought: ...
Action: ...
Action Input: ...
```

---

## 📌 防循环设计

```text
如果连续2次调用工具仍未获得有效信息，请停止并回答：
“未找到相关信息”
```

---

# 🔍 六、RAG + Agent 融合 Prompt（生产级）

---

```text
你是一个企业知识库助手。

规则：

1. 优先使用知识库（RAG）
2. 不允许编造
3. 如果检索结果不完整，可以再次检索
4. 最多检索 2 次

决策流程：

- 判断是否需要检索
- 如果需要 → 调用工具
- 如果结果足够 → 回答
- 否则 → 再检索或结束
```

---

# 🧠 七、高级技巧（关键）

---

## 🔥 7.1 Self-Check（自检机制）

```text
在输出前进行校验
```

---

### 示例

```text
请检查你的答案是否完全基于上下文：
- 是 → 输出
- 否 → 返回“无法回答”
```

---

---

## 🔥 7.2 双模型验证（企业常用）

```text
模型A生成 → 模型B校验
```

---

## 🔥 7.3 置信度输出

```json
{
  "answer": "...",
  "confidence": "low"
}
```

---

## 🔥 7.4 拒答机制

```text
不确定就拒绝，而不是胡说
```

---

# ⚠️ 八、常见错误（一定要避免）

---

## ❌ 1. Prompt 太模糊

```text
“请回答问题”
```

👉 ❌ 无约束

---

## ❌ 2. 没有限制数据来源

```text
模型会乱用知识
```

---

## ❌ 3. 没有失败路径

```text
必须设计：
“答不了怎么办？”
```

---

## ❌ 4. 输出不结构化

```text
导致不可解析
```

# 🏁 九、终极模板（建议直接用）

---

```text
你是一个严格遵循规则的AI助手。

规则：

1. 只能使用提供的信息回答
2. 不允许编造或猜测
3. 如果无法回答，必须明确说明
4. 输出必须结构化

上下文：
{context}

问题：
{question}

请输出：

{
  "answer": "...",
  "confidence": "...",
  "source": "..."
}
```

---

# 🧠 十、终极理解

---

```text
Prompt 的本质：

不是“问问题”  
而是“写程序”
```

---

```text
企业级 Prompt = 规则 + 约束 + 流程控制
```

---

```text
防幻觉的关键：

不是让模型更聪明  
而是让它“不敢乱说”
```
