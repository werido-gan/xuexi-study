# 📒 Agentic RAG 学习笔记（完整版）

---

# 🧠 一、传统 RAG

## 📌 1.1 核心流程

```
文档 → 切片 → 向量化 → 存储
                ↓
用户问题 → 向量检索 → 拼接 Prompt → LLM → 输出
```

---

## 📌 1.2 离线数据处理（Indexing）

```
数据源（PDF / 网页 / JSON）
        ↓
Load（加载）
        ↓
Transform（清洗/切片）
        ↓
Embedding（向量化）
        ↓
Vector Store（向量库）
        ↓
Document DB（原文存储）
```

---

## 📌 1.3 在线查询（Retrieval）

```
用户问题
   ↓
Query Rewrite（可选）
   ↓
Embedding(query)
   ↓
向量检索（Top-K）
   ↓
Rerank（重排）
   ↓
拼接 Prompt
   ↓
LLM
   ↓
输出结果
```

---

## 📌 1.4 核心技术

### 向量化

```
文本 → 向量（如：[0.15, 0.85, ...]）
```

### 相似度计算

```
Cosine Similarity（余弦相似度）
```

👉 用于找最相关内容

---

## ❌ 1.5 传统 RAG 问题

* 单次检索（One-shot）
* 无法纠错
* 无决策能力
* 强依赖检索质量
* 无法处理复杂任务

---

# 🚀 二、Agentic RAG

## 📌 2.1 定义

```
Agentic RAG = RAG + Agent（决策能力）
```

👉 本质：

```
把“检索”变成一个工具（Tool）
```

---

## 📌 2.2 核心思想

```
Search Ability = Tool
```

👉 模型可以：

* 什么时候查
* 查什么
* 查几次
* 是否可信

---

# 🔁 三、核心机制（重点）

## 📌 3.1 Agent 循环

```
Thought → Action → Observation → Thought
```

---

## 📌 3.2 流程图

```
User
 ↓
Query
 ↓
LLM（Agent）
 ↓
是否需要工具？
 ↓           ↓
否           是
 ↓           ↓
直接回答     调用工具（RAG / API / MCP）
                ↓
            Tool Result
                ↓
        判断是否足够
                ↓
        不够 → 继续循环
                ↓
             Response
```

# 🔥 五、对比总结

| 能力   | 传统 RAG | Agentic RAG |
| ---- | ------ | ----------- |
| 检索次数 | 1次     | 多次          |
| 是否决策 | ❌      | ✅           |
| 是否纠错 | ❌      | ✅           |
| 工具调用 | ❌      | ✅           |
| 多步推理 | ❌      | ✅           |

---

## 📌 本质区别

```
传统 RAG：检索增强
Agentic RAG：决策增强
```

---

## 📌 一句话

```
传统：查一次就回答
Agentic：边想边查，直到确认正确
```

---

# 🧪 六、运行流程（重点）

```
1. 用户输入 Query
2. LLM 生成 Thought
3. 判断是否需要工具
4. 调用工具（RAG / API）
5. 获取结果（Observation）
6. 判断是否足够
7. 不够 → 继续循环
8. 输出最终答案
```

---

# ⚠️ 八、问题

## 成本高

```
多轮推理 → Token暴涨
```

## 延迟高

```
多次调用 → 慢
```

## 不稳定

```
- 死循环
- 工具乱调用
- 错误决策
```
