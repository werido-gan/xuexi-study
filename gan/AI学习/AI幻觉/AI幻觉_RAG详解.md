# RAG（检索增强生成）：目前最有效的技术方案

> 📌 来源：OpenAI、知乎/量子位 RAG 2025年最新综述

---

## 什么是 RAG？

### 比喻理解

**普通 AI** = 靠背诵考试的学生  
→ 脑子里记了很多，但记的可能是旧的、不完整的、甚至错的

**RAG** = 可以带参考书考试的学生  
→ 遇到问题先查书，再用自己的理解给出答案，答案有据可查

---

## RAG 的工作流程

```
用户提问
    │
    ▼
1. 查询编码
   把问题转换成向量（数学表示）
    │
    ▼
2. 文档检索
   在知识库里找最相关的文段
   （就像搜索引擎）
    │
    ▼
3. 内容融合
   把检索到的证据 + 原始问题一起发给 LLM
    │
    ▼
4. 生成答案
   LLM 基于真实证据生成回答，可以附上来源
    │
    ▼
用户得到有据可查的答案 ✅
```

---

## RAG 的优势 vs 局限

| 优势 | 局限 |
|------|------|
| 平均降低 71% 的幻觉 | 需要维护知识库 |
| 知识可随时更新 | 增加延迟和计算成本 |
| 答案有来源可追溯 | 如果检索内容本身有误，仍可能出错 |
| 不需要重新训练模型 | 构建高质量知识库成本高 |

---

## 2025 年 RAG 的前沿进化

### Self-RAG（自我反思 RAG）

模型在给出答案前，先评估"我检索到的这个信息真的相关吗？"，再决定是否使用。

**效果**：进一步降低 52% 的幻觉率。

---

### Agentic RAG（智能体 RAG）

引入 AI 智能体作为协调者，可以：
- 动态决定是否需要检索
- 多轮迭代检索直到信息充足
- 自动验证答案与来源的一致性

---

### DeepNote（清华提出，2025）

模拟人类"边查边记"的思维方式，用"笔记"引导检索，比传统 RAG 性能提升约 20%。
