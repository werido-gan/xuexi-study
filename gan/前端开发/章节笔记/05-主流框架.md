## 5. 主流框架（深入）

### 5.1 React 深入解析

#### React 核心思想

**类比**：React 把页面看成"状态的函数"。UI = f(state)。就像自动贩卖机：投入不同的钱（state），输出不同的商品（UI）。状态变了，UI 自动更新。

```
[React 核心模型]

State（数据）
    │
    ▼
Component（组件函数）
    │
    ▼
Virtual DOM（虚拟DOM，JS对象描述的DOM）
    │
    ▼  Diff 算法比较新旧Virtual DOM
Real DOM（真实DOM）

优势：减少直接操作真实DOM（代价高），只更新变化的部分
```

#### React Hooks 全解

**为什么 Hooks 出现？**

Class 组件的问题：
- 生命周期分散逻辑（componentDidMount/componentDidUpdate 里可能有相关逻辑）
- `this` 让 JavaScript 初学者困惑
- 高阶组件（HOC）和 Render Props 导致组件嵌套地狱

**核心 Hooks：**

```jsx
import { useState, useEffect, useCallback, useMemo, useRef, 
         useContext, useReducer } from 'react';

// === useState: 状态管理 ===
function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <button onClick={() => setCount(prev => prev + 1)}>
      Count: {count}
    </button>
  );
}

// === useEffect: 副作用（数据获取、订阅、DOM操作）===
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  
  useEffect(() => {
    // 组件挂载 或 userId变化时执行
    let cancelled = false;
    
    fetchUser(userId).then(data => {
      if (!cancelled) setUser(data);  // 防止竞态条件
    });
    
    return () => {                    // 清理函数（组件卸载时执行）
      cancelled = true;
    };
  }, [userId]);                       // 依赖数组：userId变化才重新执行
  
  if (!user) return <div>Loading...</div>;
  return <div>{user.name}</div>;
}

// === useCallback: 缓存函数引用，防止子组件不必要重渲染 ===
function Parent() {
  const [count, setCount] = useState(0);
  
  // 没有useCallback：每次Parent渲染，handleClick都是新函数
  // 导致依赖它的子组件重渲染
  const handleClick = useCallback(() => {
    console.log('clicked');
  }, []);  // 依赖为空，函数永远是同一个引用
  
  return <Child onClick={handleClick} />;
}

// === useMemo: 缓存计算结果 ===
function ExpensiveComponent({ data }) {
  // 每次渲染都重新计算 → 浪费
  // const result = expensiveCalculation(data);
  
  // 只有 data 变化时才重新计算
  const result = useMemo(() => expensiveCalculation(data), [data]);
  
  return <div>{result}</div>;
}

// === useRef: 获取DOM引用 或 存储不触发重渲染的值 ===
function TextInput() {
  const inputRef = useRef(null);
  
  const focusInput = () => {
    inputRef.current.focus();  // 直接操作DOM
  };
  
  // 存储上一次的值（不触发重渲染）
  const prevCountRef = useRef(0);
  useEffect(() => {
    prevCountRef.current = count;
  });
  
  return <input ref={inputRef} />;
}

// === useReducer: 复杂状态管理 ===
const initialState = { count: 0, error: null };

function reducer(state, action) {
  switch (action.type) {
    case 'INCREMENT': return { ...state, count: state.count + 1 };
    case 'DECREMENT': return { ...state, count: state.count - 1 };
    case 'RESET':     return initialState;
    default: throw new Error(`Unknown action: ${action.type}`);
  }
}

function Counter() {
  const [state, dispatch] = useReducer(reducer, initialState);
  
  return (
    <>
      <p>Count: {state.count}</p>
      <button onClick={() => dispatch({ type: 'INCREMENT' })}>+</button>
      <button onClick={() => dispatch({ type: 'DECREMENT' })}>-</button>
    </>
  );
}
```

**自定义 Hooks（复用逻辑的最佳方式）：**

```jsx
// 自定义 Hook：数据获取
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  
  useEffect(() => {
    setLoading(true);
    fetch(url)
      .then(res => res.json())
      .then(data => {
        setData(data);
        setLoading(false);
      })
      .catch(err => {
        setError(err);
        setLoading(false);
      });
  }, [url]);
  
  return { data, loading, error };
}

// 使用
function UserList() {
  const { data: users, loading, error } = useFetch('/api/users');
  
  if (loading) return <Spinner />;
  if (error) return <Error message={error.message} />;
  return <ul>{users.map(u => <li key={u.id}>{u.name}</li>)}</ul>;
}
```

#### React Fiber 架构

**为什么需要 Fiber？**

React 15 的问题：虚拟 DOM 的 Diff 和更新是**同步递归**的，一旦开始就无法中断。如果组件树很大，会阻塞主线程，导致页面卡顿（掉帧）。

```
[React 15 Stack Reconciler 问题]

Diff 开始
  └── 递归整棵树（不可中断）
       └── 如果树很大，需要100ms
           └── 用户点击、动画 → 全部卡住

[React 16+ Fiber Reconciler 解决方案]

Fiber = 将工作分解成小单元（每个组件是一个Fiber节点）
     = 支持暂停、恢复、中断

渲染分两阶段：
1. Reconciliation（协调，可中断）
   └── 构建 Fiber 树
   └── 计算哪些需要更新
   └── 利用浏览器空闲时间（requestIdleCallback思想）

2. Commit（提交，不可中断）
   └── 将变更应用到真实DOM
   └── 执行生命周期/副作用
```

### 5.2 Vue 响应式原理

#### Vue 2 vs Vue 3 响应式对比

```
[Vue 2 响应式原理（Object.defineProperty）]

data: { count: 0 }
    │
    ▼  Vue2 初始化时遍历所有属性
Object.defineProperty(obj, 'count', {
  get() { 收集依赖（Watcher） },
  set(newVal) { 通知更新（Dep.notify） }
})

问题：
1. 新增属性无法检测（需要 Vue.set）
2. 数组下标修改无法检测（需要 splice 等）
3. 初始化需要递归遍历所有属性，性能有损耗

[Vue 3 响应式原理（Proxy）]

const state = reactive({ count: 0 })
    │
    ▼  使用 ES6 Proxy 代理整个对象
new Proxy(target, {
  get(target, key) { 
    track(target, key)  // 收集依赖
    return target[key] 
  },
  set(target, key, value) {
    target[key] = value
    trigger(target, key) // 触发更新
    return true
  }
})

优势：
1. 可以检测新增属性
2. 可以检测数组下标变化
3. 懒代理：访问时才代理，性能更好
```

#### Vue 3 Composition API

```javascript
// Vue 3 组合式 API
import { ref, reactive, computed, watch, onMounted } from 'vue';

export default {
  setup() {
    // ref: 基本类型响应式
    const count = ref(0);
    
    // reactive: 对象响应式
    const user = reactive({
      name: '张三',
      age: 25
    });
    
    // computed: 计算属性
    const doubleCount = computed(() => count.value * 2);
    
    // watch: 监听变化
    watch(count, (newVal, oldVal) => {
      console.log(`count: ${oldVal} → ${newVal}`);
    });
    
    // 生命周期
    onMounted(() => {
      console.log('组件已挂载');
    });
    
    const increment = () => count.value++;
    
    return { count, user, doubleCount, increment };
  }
};

// 使用 <script setup>（更简洁，推荐）
<script setup>
import { ref, computed } from 'vue';

const count = ref(0);
const doubleCount = computed(() => count.value * 2);
const increment = () => count.value++;
</script>
```

### 5.3 React vs Vue 对比

| 维度 | React | Vue |
|-----|-------|-----|
| 学习曲线 | 较陡（需要理解JSX、Hooks原理） | 较平缓（有完善的中文文档） |
| 灵活性 | 高（只是UI库，其他自选） | 中（有官方推荐方案） |
| 性能 | 优秀（Fiber调度） | 优秀（响应式更精细） |
| 生态 | 极其丰富（Meta背书） | 丰富（阿里、腾讯大量使用） |
| 适用场景 | 大型复杂应用，团队规模大 | 中小型项目，快速开发 |
| TypeScript | 支持良好 | Vue3原生TS |
| 中国流行度 | 高 | 非常高（国内领先） |

### 5.4 第五章总结

```
[主流框架核心对比]

框架选择
├── React
│   ├── 核心: UI = f(state)
│   ├── Hooks: useState/useEffect/useCallback
│   ├── Fiber: 可中断的异步渲染
│   └── 适合: 大型团队，复杂应用
│
└── Vue
    ├── 核心: 响应式数据驱动
    ├── Vue2: Object.defineProperty
    ├── Vue3: Proxy（更强大）
    └── 适合: 快速开发，中小团队
```
