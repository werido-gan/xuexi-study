## 9. 全栈能力拓展

### 9.1 Node.js

**Node.js 是什么？**

把 Chrome 的 V8 引擎拿出来，加上文件系统、网络等能力，让 JavaScript 可以在服务器端运行。

```
[Node.js 架构]

JavaScript代码
     │
     ▼
V8引擎（JS执行）
     │
     ▼
Node.js Bindings（C++桥接层）
     │
     ▼
libuv（跨平台异步I/O库）
├── 事件循环
├── 线程池（文件I/O等阻塞操作）
└── 异步网络I/O
```

**Express 快速上手：**

```javascript
const express = require('express');
const app = express();

// 中间件
app.use(express.json());             // 解析JSON请求体
app.use(express.static('public'));   // 静态文件服务
app.use(cors());                     // 跨域处理

// RESTful API 路由
app.get('/api/users', async (req, res) => {
  try {
    const { page = 1, limit = 10 } = req.query;
    const users = await User.find()
      .skip((page - 1) * limit)
      .limit(limit);
    res.json({ data: users, page, total: await User.countDocuments() });
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});

app.post('/api/users', async (req, res) => {
  const { name, email, password } = req.body;
  const user = new User({ name, email, password: await hash(password) });
  await user.save();
  res.status(201).json(user);
});

app.listen(3000, () => console.log('Server running on port 3000'));
```

### 9.2 RESTful API 设计原则

```
[RESTful API 设计规范]

资源用名词，动作用HTTP方法:
GET    /api/products          → 获取商品列表
GET    /api/products/:id      → 获取单个商品
POST   /api/products          → 创建商品
PUT    /api/products/:id      → 更新商品（全量）
PATCH  /api/products/:id      → 更新商品（部分）
DELETE /api/products/:id      → 删除商品

版本控制:
/api/v1/products
/api/v2/products

统一响应格式:
{
  "code": 0,              // 0 成功，非0 失败
  "message": "success",
  "data": { ... },        // 实际数据
  "pagination": {         // 分页信息（列表接口）
    "page": 1,
    "pageSize": 10,
    "total": 100
  }
}
```

### 9.3 数据库基础

```
[关系型 vs 非关系型数据库]

关系型（SQL）: MySQL / PostgreSQL
  ├── 数据结构化，表格形式
  ├── 强 ACID 事务
  ├── 适合：订单、用户、金融数据
  └── 缺点：水平扩展困难

非关系型（NoSQL）: MongoDB / Redis
  ├── MongoDB: 文档型，灵活的JSON结构
  │   适合：内容管理，商品目录
  ├── Redis: 内存键值型，极速
  │   适合：缓存、Session、消息队列
  └── 优点：水平扩展容易
```
