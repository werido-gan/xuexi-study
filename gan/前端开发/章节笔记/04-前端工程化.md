## 4. 前端工程化（重点）

### 4.1 包管理工具

**为什么需要包管理？**

想象你要建一栋楼，需要砖、水泥、钢筋，你不需要自己生产这些材料，去建材市场（npm）买就行。包管理工具就是帮你管理"建材"（第三方库）的工具。

#### npm（Node Package Manager）

```bash
# 初始化项目
npm init -y

# 安装依赖
npm install react                    # 生产依赖
npm install webpack --save-dev       # 开发依赖（--save-dev 简写 -D）
npm install -g typescript            # 全局安装

# package.json 中的依赖分类
{
  "dependencies": {               // 生产环境需要
    "react": "^18.2.0",
    "axios": "^1.4.0"
  },
  "devDependencies": {           // 仅开发时需要
    "webpack": "^5.88.0",
    "typescript": "^5.1.6"
  }
}

# 版本号规则（语义化版本 Semver）
"^18.2.0"  → 兼容 18.x.x（主版本不变，次版本可更新）
"~18.2.0"  → 兼容 18.2.x（仅补丁版本可更新）
"18.2.0"   → 精确版本，不允许更新
```

#### pnpm（Performance npm，推荐！）

**为什么选 pnpm？**

```
[npm/yarn 的问题]

项目A: node_modules/
  └── react/ (50MB)
  └── lodash/ (10MB)

项目B: node_modules/
  └── react/ (50MB)  ← 重复存储！
  └── lodash/ (10MB) ← 重复存储！

总计: 120MB

[pnpm 的解决方案]

全局存储（~/.pnpm-store）
  └── react@18.2.0/  ← 只存一份
  └── lodash@4.17.21/ ← 只存一份

项目A/node_modules/react → 硬链接 → 全局存储
项目B/node_modules/react → 硬链接 → 全局存储（同一份）

总计: 60MB，节省50%！
```

```bash
# pnpm 常用命令
pnpm install          # 安装所有依赖
pnpm add react        # 添加依赖
pnpm add -D webpack   # 添加开发依赖
pnpm run dev          # 运行脚本
pnpm dlx create-react-app  # 执行一次性命令（类似npx）
```

### 4.2 构建工具：Webpack vs Vite

#### Webpack（老牌构建工具）

**核心概念：**

```
[Webpack 构建流程]

入口文件（entry）
    │
    ▼
依赖图（Dependency Graph）
    │  Webpack递归分析import/require
    │  找到所有依赖文件
    │
    ▼
Loader（转换器）
    │  处理各种文件类型
    │  babel-loader: JS/JSX → ES5
    │  css-loader: CSS → JS模块
    │  file-loader: 图片 → URL
    │
    ▼
Plugin（插件）
    │  HtmlWebpackPlugin: 生成HTML
    │  MiniCssExtractPlugin: 提取CSS
    │  TerserPlugin: 压缩JS
    │
    ▼
输出文件（output）
    dist/
    ├── bundle.js
    ├── style.css
    └── index.html
```

```javascript
// webpack.config.js 基础配置
const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
  entry: './src/index.js',           // 入口
  
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: '[name].[contenthash].js',  // contenthash 用于缓存
    clean: true                           // 每次构建清理dist
  },
  
  module: {
    rules: [
      {
        test: /\.(js|jsx)$/,             // 匹配文件类型
        use: 'babel-loader',             // 使用哪个Loader
        exclude: /node_modules/
      },
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader']  // 从右到左执行
      },
      {
        test: /\.(png|jpg|gif)$/,
        type: 'asset/resource'           // Webpack5内置资源处理
      }
    ]
  },
  
  plugins: [
    new HtmlWebpackPlugin({
      template: './public/index.html'
    })
  ],
  
  optimization: {
    splitChunks: {
      chunks: 'all',                    // 代码分割：提取公共模块
    }
  }
};
```

#### Vite（现代构建工具，推荐新项目使用）

**为什么 Vite 比 Webpack 快？**

```
[Webpack 开发模式]

启动时：分析所有文件 → 打包成bundle → 启动服务器
修改代码：重新打包整个bundle（文件多时非常慢）

[Vite 开发模式]

启动时：直接启动服务器（极速！）
修改代码：只更新修改的模块（HMR）

原理：浏览器原生支持ES Module，Vite直接让浏览器加载源文件
不需要打包！开发时实现"按需编译"

[Vite 为什么快？]

传统打包：
源码 → Webpack打包 → bundle.js → 浏览器

Vite方式：
源码 → Vite服务器 → 浏览器直接请求各个模块 → 按需编译
```

```javascript
// vite.config.js
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins: [react()],
  
  server: {
    port: 3000,
    proxy: {
      '/api': {                          // 开发时代理API，解决跨域
        target: 'http://localhost:8080',
        changeOrigin: true
      }
    }
  },
  
  build: {
    rollupOptions: {
      output: {
        manualChunks: {                  // 手动代码分割
          vendor: ['react', 'react-dom'],
          utils: ['lodash', 'axios']
        }
      }
    }
  }
});
```

### 4.3 Babel 原理

**类比**：Babel 就是英语翻译官，把你写的"现代英语"（ES2022+语法）翻译成"老式英语"（ES5），让不支持新语法的老浏览器也能理解。

```
[Babel 编译流程]

源代码（ES2022）
    │
    ▼  解析（Parse）
AST（抽象语法树）
    │  Abstract Syntax Tree
    │  将代码转成树形数据结构
    │
    ▼  转换（Transform）
新的 AST（ES5语法的AST）
    │  各种 Plugin/Preset 在这里工作
    │
    ▼  生成（Generate）
目标代码（ES5）

[AST示例]
// 源代码
const add = (a, b) => a + b;

// AST（简化）
{
  type: "VariableDeclaration",
  kind: "const",
  declarations: [{
    type: "VariableDeclarator",
    id: { type: "Identifier", name: "add" },
    init: {
      type: "ArrowFunctionExpression",
      params: [{ name: "a" }, { name: "b" }],
      body: { type: "BinaryExpression", operator: "+" }
    }
  }]
}
```

```json
// .babelrc 配置
{
  "presets": [
    ["@babel/preset-env", {
      "targets": {
        "browsers": ["> 1%", "last 2 versions"]
      },
      "useBuiltIns": "usage",    // 按需引入 polyfill
      "corejs": 3
    }],
    "@babel/preset-react",       // 转换 JSX
    "@babel/preset-typescript"   // 转换 TypeScript
  ],
  "plugins": [
    "@babel/plugin-proposal-decorators"  // 装饰器支持
  ]
}
```

### 4.4 代码规范

#### ESLint（代码质量检查）

```javascript
// .eslintrc.js
module.exports = {
  extends: [
    'eslint:recommended',
    'plugin:react/recommended',
    'plugin:@typescript-eslint/recommended',
    'prettier'               // 关闭与Prettier冲突的规则
  ],
  rules: {
    'no-console': 'warn',   // 生产代码禁止console
    'no-unused-vars': 'error',
    'react/prop-types': 'off',  // TypeScript项目不需要
    'prefer-const': 'error',    // 优先使用const
  }
};
```

#### Prettier（代码格式化）

```json
// .prettierrc
{
  "semi": true,              // 分号
  "singleQuote": true,       // 单引号
  "tabWidth": 2,             // 缩进
  "trailingComma": "all",    // 尾逗号
  "printWidth": 100          // 行宽
}
```

> **最佳实践**：结合 `husky` + `lint-staged`，在每次 `git commit` 时自动运行 ESLint 和 Prettier，保证提交到仓库的代码都是规范的。

### 4.5 Git 工作流

#### Git Flow（适合有版本发布概念的项目）

```
[Git Flow 分支模型]

main（主分支，生产环境）
  │
  ├─── develop（开发主分支）
  │       │
  │       ├─── feature/login（功能分支）
  │       │     └── 完成后 merge 回 develop
  │       │
  │       ├─── feature/payment
  │       │
  │       └─── release/v1.2.0（发布分支）
  │             └── 测试通过后 merge 到 main 和 develop
  │
  └─── hotfix/critical-bug（紧急修复）
        └── 修复后 merge 到 main 和 develop
```

#### Trunk-Based Development（适合 CI/CD 频繁发布）

```
[Trunk-Based Development]

main（主干，所有人都 push 到这里）
  │
  ├── feature flag 控制功能开关
  ├── 小批量提交（每天至少一次）
  └── 自动化测试保证质量
```

**常用 Git 命令速查：**

```bash
# 基础操作
git clone <url>              # 克隆仓库
git checkout -b feature/xxx  # 创建并切换分支
git add .                    # 暂存所有修改
git commit -m "feat: 添加登录功能"  # 提交（遵守Conventional Commits规范）
git push origin feature/xxx  # 推送到远程

# Commit 规范（Conventional Commits）
feat:     新功能
fix:      修复Bug
docs:     文档变更
style:    代码格式（不影响功能）
refactor: 重构
perf:     性能优化
test:     添加测试
chore:    构建工具变更

# 实用命令
git stash          # 临时保存修改
git stash pop      # 恢复暂存
git rebase -i HEAD~3  # 合并最近3个提交
git cherry-pick <hash>  # 把某个提交应用到当前分支
```
